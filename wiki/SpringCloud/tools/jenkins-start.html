JK的简单使用

安装

如此如此, 这般这般…

Gitee

安装插件

插件搜索 gitee, 只有那一个.

配置令牌

去到 系统管理 -> 系统配置 -> Gitee配置

  1. 链接名: Gitee
  2. Gitee 域名 URL: https://gitee.com
  3. 添加 -> 类型选择 -> Gitee API 令牌
    • 申请去 这里
  4. 其它默认, 点击添加
  5. 测试链接, 然后保存.

创建任务

创建:

  • 任务名称: 随便
  • 构建一个自由风格的软件项目

General:

  • Gitee链接: Gitee(就是刚才的链接名: Gitee)
  • 勾选丢弃旧的构建
    • 保持天数: 1
    • 保持个数: 3

源码管理:

  • 选择 Git
  • 填写仓库地址
  • 添加账号密码
    • 类型为 UserName With PassWord
    • 用户名: Gitee账号
    • 密码: Gitee密码
    • 描述: Gitee-TC-ArtMarket-Services的构建账号

构建触发器:

  • 选择 Gitee webhook
  • 勾选的那行有个链接记下来
  • Gitee WebHook 密码: 点击生成, 和链接一起使用

构建:
看你自己项目需要咯, 先不写吧

Gitee配置

  • 来到仓库设置 -> WebHook
  • 新建一个, 填上链接和密码即可

Gitee企业

企业版的仓库好像要收费走工作流, 克隆到个人仓库添加WebHook是可以的, 给克隆仓库同步企业仓库更新就行.

全局环境

工作目录: 在docker安装挂载的目录

配置环境:

  • 就和普通linux配置一样, 在工作目录准备一个文件夹
  • 把需要的jdk、maven、node等下载解压配置好
  • 然后在jk容器的命令行暴露为系统环境
  • 最后在jk的网页上, 系统设置 -> 全局配置里, 按/var/jenkins_home/xxxxx去填写到对应的服务配置上就行

话是这么说很简单, 但操作挺繁琐的.
当然已经到了 Jenkins 这步, 配环境对你来说已经像吃饭一样了吧

测试配置:

  • 在已经建好的任务的Build Steps添加一个shell测试一下
  • 保存出来点击构建, 看有没有打印对应的内容
